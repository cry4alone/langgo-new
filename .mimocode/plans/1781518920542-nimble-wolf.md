# План: Система приглашений в игру

## Концепция

Пользователь в лобби (создал игру, ждёт игроков) может пригласить конкретного игрока через NotificationHub. Приглашённый может принять или отклонить. При принятии — автоматически присоединяется к комнате через GameHub.

## Флоу

```
1. Host → POST /games/rooms/{roomId}/invite { userId }
   → Валидация: игра существует, Status=Waiting, host=текущий пользователь, есть место
   → Отправка GameInviteReceived через NotificationHub приглашённому

2. Invited → получает событие GameInviteReceived { roomId, hostUsername, gameSettings }

3. Invited → POST /games/rooms/{roomId}/invite/respond { accept: true/false }
   → Валидация: игра существует, Status=Waiting, есть место
   → Если true: возвращаем 200, клиент вызывает GameHub.JoinRoom(roomId) для входа
   → Если false: отправляем GameInviteDeclined через NotificationHub хосту
```

## События NotificationHub

| Событие | Кому | Payload |
|---|---|---|
| `GameInviteReceived` | приглашённый | `{ roomId, hostUserId, hostUsername, gameSettings }` |
| `GameInviteDeclined` | хост | `{ roomId, invitedUserId }` |

## Валидация

- Игра должна существовать и быть в Redis (GameState)
- Status должен быть `Waiting`
- Вызывающий должен быть `HostUserId`
- В комнате должно быть место (max 4 игрока: 1 host + 3 в PlayerUserIds)
- Нельзя пригласить себя
- Нельзя пригласить уже присоединённого игрока
- Нельзя пригласить игрока, которому уже отправлен инвайт

## Клиентская логика (после принятия)

После успешного `POST /invite/respond { accept: true }`, клиент должен:
1. Подключиться к GameHub (если ещё не подключён)
2. Вызвать `GameHub.JoinRoom(roomId)`
3. Получить `ReceiveRoomState` с обновлённым списком игроков

## Верификация

1. `dotnet build` — 0 ошибок
2. Тест через test-notifications.html:
   - A создаёт игру → получает RoomId
   - A приглашает B через POST /games/rooms/{roomId}/invite
   - B получает `GameInviteReceived` в NotificationHub
   - B принимает через POST /games/rooms/{roomId}/invite/respond
   - B подключается к GameHub и вызывает JoinRoom(roomId)
   - A видит `PlayerJoined` и обновлённый список игроков
